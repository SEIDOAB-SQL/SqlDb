# Code-First Database with Entity Framework Core — Helicopter View

## 1. Configure the Database Connection

Set up the connection string in `appsettings.json` (or better user secrets) 

---

## 2. Define the Model Classes

Create a C# class for each entity in the `Models` project. Each class maps to one database table.

```csharp
public class Friend
{
    public Guid FriendId { get; set; }   // primary key
    public string FirstName { get; set; }
    public string LastName  { get; set; }
}
```

---

## 3. Set the Primary Keys

EF Core recognises `Id` or `<ClassName>Id` as the primary key by convention. For any other property name, use the `[Key]` data annotation or the Fluent API:

```csharp
[Key]
public Guid FriendId { get; set; }
```

---

## 4. Add Navigation Properties for Relationships

Add navigation properties to **all related classes** to represent the cardinality of each relationship:

| Relationship | Navigation property |
|---|---|
| One-to-many (parent side) | `public List<Address> Addresses { get; set; }` |
| Many-to-one (child side) | `public Friend Friend { get; set; }` |
| Many-to-many | `public List<MusicGroup> MusicGroups { get; set; }` on both sides |

EF Core uses these properties to infer foreign keys and generate the correct join tables.

---

## 5. Register the Models in the DbContext

In `MainDbContext.cs`, add a `DbSet<T>` property for every entity that should map to a table:

```csharp
public class MainDbContext : DbContext
{
    public DbSet<Friend>  Friends  { get; set; }
    public DbSet<Address> Addresses { get; set; }
    public DbSet<Pet>     Pets     { get; set; }
}
```

---

## 6. Build the Project

Compile the solution and verify there are no errors before running any EF Core tooling commands.

```bash
dotnet build
```

---

## 7. Create a Migration

From a terminal, navigate to the `DbContext` project folder and generate a migration. EF Core compares the current model state against the last migration snapshot and produces the up/down SQL scripts:

```bash
dotnet ef migrations add InitialCreate
```

Review the generated files in the `Migrations/` folder before applying them.

---

## 8. Apply the Migration to the Database

Push the migration to the target database. EF Core runs the generated SQL and records the migration in the `__EFMigrationsHistory` table:

```bash
dotnet ef database update
```

The database now reflects the current state of the model classes.
